=================================================
  AirBar Installer - Guide de Test Complet
=================================================

Version: 1.0.0
Date: Avril 2026

Ce document décrit la procédure complète de test de
l'application AirBar Installer.

=================================================
PRÉPARATION DE L'ENVIRONNEMENT DE TEST
=================================================

1. Machine de Test
   - Windows 10/11 64-bit (idéalement VIERGE)
   - 8 GB RAM minimum
   - 20 GB d'espace disque libre
   - Connexion Internet stable
   
2. Compte Utilisateur
   - Privilèges Administrateur requis
   
3. Nettoyage Préliminaire (si tests antérieurs)
   □ Désinstaller Docker Desktop (si présent)
   □ Désinstaller Git (si présent)
   □ Supprimer C:\AirBar (si présent)
   □ Vérifier qu'aucun conteneur Docker ne tourne
   □ Nettoyer les règles de pare-feu AirBar

=================================================
PHASE 1: TEST DE BUILD
=================================================

Test 1.1: Vérification du Projet
--------------------------------
□ Cloner le repository airbar_installer
□ Ouvrir un terminal PowerShell
□ Exécuter: flutter doctor
   ✓ Attendu: Aucun problème critique
□ Exécuter: flutter pub get
   ✓ Attendu: Dépendances téléchargées sans erreur

Test 1.2: Analyse du Code
--------------------------------
□ Exécuter: flutter analyze
   ✓ Attendu: 0 erreurs (avertissements OK)
   
Test 1.3: Build Debug
--------------------------------
□ Exécuter: flutter build windows --debug
   ✓ Attendu: Build réussit
   ✓ Vérifier: build\windows\x64\runner\Debug\airbar_installer.exe existe

Test 1.4: Build Release
--------------------------------
□ Exécuter: flutter build windows --release
   ✓ Attendu: Build réussit sans erreurs
   ✓ Vérifier: build\windows\x64\runner\Release\airbar_installer.exe existe
   ✓ Vérifier: Taille > 10 MB (environ 15-30 MB)
   ✓ Vérifier: Dossier data\ présent

Test 1.5: Build avec Script Automatique
--------------------------------
□ Créer l'icône app_icon.ico (ou copier depuis windows\runner\resources)
□ Placer dans assets\images\app_icon.ico
□ Exécuter: .\build.ps1
   ✓ Attendu: 
     - Analyse passe
     - Build Windows réussit
     - Fichiers vérifiés
     - Inno Setup compile (si installé)
   ✓ Vérifier: build\windows\installer\*.exe créé (si Inno Setup)

PHASE 3: TEST DE L'ASSISTANT D'INSTALLATION
=================================================

Test 3.1: Écran Splash
--------------------------------
□ Application démarre
   ✓ Attendu: Logo AirBar affiché 2-3 secondes
   ✓ Transition automatique vers Welcome

Test 3.2: Écran Welcome
--------------------------------
□ Vérifier: Texte de bienvenue affiché
□ Vérifier: Bouton "Commencer" visible
□ Cliquer: Commencer
   ✓ Transition vers System Check

Test 3.3: Écran System Check
--------------------------------
□ Observer: Vérifications en cours
   ✓ Windows Version
   ✓ Architecture 64-bit
   ✓ RAM disponible
   ✓ Espace disque
   ✓ PowerShell version
   ✓ Docker présent/absent
   ✓ Git présent/absent
□ Si Docker ABSENT:
   ✓ Attendu: Message "Docker non détecté"

□ Si Docker PRÉSENT:

□ Vérifier: Bouton "Suivant" activé

=================================================
PROBLÈMES CONNUS ET SOLUTIONS
=================================================

Problème: Docker ne démarre pas après installation
Solution: Redémarrer Windows

Problème: Ports déjà utilisés
Solution: Modifier dans Config ou arrêter l'autre application

Problème: Permission denied lors du build
Solution: Désactiver antivirus temporairement

Problème: Git clone échoue
Solution: Vérifier connexion Internet, proxy
